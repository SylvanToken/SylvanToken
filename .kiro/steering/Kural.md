# Project Rules

## Language Requirements
- All code and documentation must be written in English.
- Explanations to me should be provided in Turkish.

## Project Information
- Project start year: 2025
- When adding date information, use format: "2025-" (continuing)
- Example: "Date: November 8, 2025-"

## Version Control & Change Management
- A CHANGELOG.md file must be maintained at the project root
- Every file modification must be recorded in CHANGELOG.md
- Version format: Semantic versioning (e.g., 1.0.1)
- Version increments:
  - Major (1.0.0 → 2.0.0): Breaking changes
  - Minor (1.0.0 → 1.1.0): New features
  - Patch (1.0.0 → 1.0.1): Bug fixes, documentation updates
- Each deployment must increment the version number
- New files must be recorded in CHANGELOG.md
- Each changelog entry must include:
  - Version number
  - Date
  - Type of change (Added, Changed, Fixed, Removed, Security, Deprecated)
  - List of modified/added files
  - Brief description of changes
